# PKA 问答控件全量主题跟随设计

## 目标

使问答页的导出与资料操作按钮、参考来源链接、来源类型徽标和提示文本在浅色或深色主题下均使用可读的前景、背景和边框颜色；组件样式不再写入固定颜色值。

## 范围

- `static/style.css` 的问答头部操作栏、资料去向按钮、来源区域与来源徽标。
- 保持现有 DOM、交互、按钮禁用逻辑和来源类型语义不变。

## 方案

1. 在根主题中建立语义变量：次级操作、资料去向、来源链接、来源徽标和禁用状态各自具备 `foreground`、`background`、`border` 变量。
2. 组件规则只引用这些变量及通用主题变量，禁止在问答控件中使用十六进制、`rgb`、`rgba` 或指定主题色名。
3. 来源类型仍保留“组织图 / PDF / Text / Generic”的区分，但每一类均通过主题变量提供颜色；默认来源类型使用独立的可读变量，避免依赖未定义的 `--soft`。
4. 悬停、禁用与来源提示也使用相同语义层，确保主题切换后前景和背景成对变化。

## 验收

- 问答控件区域没有固定颜色字面量。
- 操作按钮、禁用按钮、来源链接、四类徽标和来源提示均引用主题变量。
- 现有问答页面结构与操作按钮状态测试继续通过。
